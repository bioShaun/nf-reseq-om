文库构建

Total 4.5 mg of genomic DNA (35 ng/ml) was fragmented into 300 bp using the Bioruptor UCD-200 sonicator (Diagenode, Denville, NJ). Pre-libraries were constructed using the Kapa Hyper DNA library prep kit for Illumina HiSeq. Fragmented DNA (1 mg) was end repaired with an end-repair enzyme and a deoxyadenosine was added to the 30 end of the fragments. Kapa barcoded DNA and Kapa indexed adapters were ligated to the sample libraries. The adapter-ligated libraries were selected for an average insert size of 350 bp using next-generation sequencing cleanup and size select (NucleoMag, Macherey-Nagel, Duren, Germany) according to the manufacturer’s instructions. The pre-capture amplification was performed using the NumbleGen protocol with eight PCR cycles. Equal amounts of products from six libraries were pooled to obtain a total of 1 mg of DNA for the hybridization. Hybridization of sample libraries was performed at 47C for 60h~72 h using the SeqCap EZ library (Roche/NimbleGen, Madison, WI), probe custom designed with IWGSC RefSeq V1.0 genome. The hybrids were captured using the capture beads from the SeqCap EZ pure capture bead kit, washed, and amplified by ligationmediated PCR. The quality of the captured libraries was assessed using the Bioanalyzer 4200 instrument (Agilent Technologies, Santa Clara, CA). The libraries were quantified by quantitative real-time (qRT)-PCR and then sequenced using the Illumina HiSeq Nova platform to generate 150-bp paired-end reads.

The SeqCap EZ HyperCap workflow (Figure 1) involves: 1. Preparation of the gDNA sample library using a KAPA Hyper Prep Kit or KAPA HyperPlus Library Preparation Kit. 2. Amplification of the DNA sample library using KAPA HiFi HotStart ReadyMix (2X). 3. Capturing target regions by hybridizing the gDNA sample library with the SeqCap EZ probe pool. 4. Recovery of captured sample using the SeqCap Hybridization and Wash Kit. 5. Amplification of the captured DNA sample using KAPA HiFi HotStart ReadyMix (2X). 6. Sequencing the captured and amplified DNA sample using an Illumina sequencing instrument.

双端测序

Paired-end方法是指在构建待测DNA文库时在两端的接头上都加上测序引物结合位点，在第一轮测序完成后，去除第一轮测序的模板链，用对读测序模块(Paired-End Module)引导互补链在原位置再生和扩增，以达到第二轮测序所用的模板量，进行第二轮互补链的合成测序。

原始序列数据

提供的文件格式一般为：项目编号/Flowcell号/样品名称_Lane号_I+index号.R1戒R2.clean.fastq.gz(例1_L5_I108.R1.clean.fastq.gz) 提供的数据为压缩包形式，数据量比较大，解压缩后一般在window下不易打开，需要在linux下用less命令打开或者用其他编辑器打开。 数据格式的详细说明参考以下的网页：http://en.wikipedia.org/wiki/FASTQ_format
FASTQ格式文件中每个read由四行描述组成，如下：

FASTQ格式文件图

第一行：以“@”开头，随后为Illumina 测序标识符(Sequence Identifiers)和index序列等；
第二行：即为测到的碱基序列；
第三行：以“+”开头，随后为Illumina 测序标识符(选择性部分)；

第四行为对应第二行碱基的测序质量值。如果测序错误率用e表示，illumina HiSeqTM2500/MiSeq的碱基质量值用Qphred表示，则有下列关系：
公式一：　Qphred　=　-10log10(e)

数据过滤

通过设定一定的过滤标准，去除掉会对后续的信息分析造成干扰的接头信息、低质量碱基、未测出的碱基（以N表示）等。最终得到的数据，我们称之为clean data 或clean reads。我们的数据过滤标准为：
1) 过滤掉含有接头序列（adaptor）的reads；
2) 当单端测序read中含有的N的含量超过该条read长度比例的3%时,去除此对pairedreads；
3) 当单端测序read中含有的低质量(低于3)碱基数超过该条read长度比例的50%时，去除此对paired reads。

测序错误率原因

对于测序技术，测序错误率分布具有下面两个主要原因：
1) 由于测序过程中化学试剂的消耗，导致测序错误率会随着测序序列(Sequenced Reads)长度的增加而升高。
2) PCR过程中随机引物和DNA模版的不完全结合(Jiang et al.)可能导致前几个碱基测序错误率较高。

主要分析流程

分析方法及软件版本

Raw sequencing reads were processed using ‘Fastap’(v0.12.4) to remove low quality read and adapter.BWA (version 0.7.16) mem was applied to align the high quality reads to IWGSC RefSeq V1.0 genome with default parameters. samtools (version 1.9) were used to bam sort, remove read PCR duplications and sample alignment statistic. After that raw cohort vcf was worked out with GATK (version v4.0.10.1) module BaseRecalibrator, ApplyBQSR, HaplotypeCaller, CombineGVCFs and GenotypeGVCFs. The minimum-mapping-quality parameter was set as 30 for only high quality alignment reads used to call variants. bcftools (version 1.9) was performed as variants quality filtering with "QUAL>30" and "DP>5". To annotate variants, customized database contained IWGSC V1.1 HC/LC gene and own annotated gene were created by snpEff.

Reads比对结果IGV可视化浏览

我们提供 Reads在基因组上比对结果的bam格式文件，部分物种还提供相应的参考基因组和注释文件，并推荐使用IGV (Integrative Genomics Viewer) 浏览器对bam文件进行可视化浏览。IGV浏览器具有以下特点：(1)能在不同尺度下显示单个或多个读段在基因组上的位置，包括读段在各个染色体上的分布情况和在注释的外显子、内含子、剪接接合区、基因间区的分布情况等；(2)能在不同尺度下显示不同区域的读段丰度，以反映不同区域的转录水平；(3)能显示基因及其剪接异构体的注释信息；(4)能显示其他注释信息；(5)既可以从远程服务器端下载各种注释信息，又可以从本地加载注释信息。 参考IGV浏览器使用方法。
